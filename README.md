# Беш Админ — приложение для администрации

Приложение для администраторов: статистика, управление заказами, разбор жалоб, список пользователей.

## Что внутри

- **Статистика** — общие показатели (заказы, выручка, пользователи)
- **Заказы** — все заказы в системе, фильтрация по статусу
- **Меню** — управление блюдами: добавление, редактирование, удаление, загрузка стартового набора
- **Жалобы** — разбор жалоб от клиентов и курьеров (бейдж с количеством непрочитанных)
- **Пользователи** — список всех пользователей с фильтрами по ролям

## Управление меню

Открыв вкладку «Меню» в первый раз (когда коллекция `dishes` в Firestore пуста),
вы увидите кнопку **«Загрузить стартовое меню»** — она зальёт 20 национальных блюд
одним нажатием. После этого можно добавлять, редактировать и удалять блюда.

Все изменения сразу видны клиентскому приложению (оно подписано на ту же
коллекцию `dishes` через `snapshots()`).

### Форма блюда

В форме добавления/редактирования есть:
- Эмодзи (отображается вместо фото)
- Название и описание
- Цена в тенге и вес в граммах
- Категория: Мясное, Тесто, Супы, Хлеб, Сладкое, Напитки
- Теги: Хит, Новинка, Национальное, Острое, Веган (можно несколько)

## Тестовый аккаунт

Вход в один клик, без ввода логина и пароля — используется специальный
демо-аккаунт администратора.

```
admin@besh.kz / admin_secret_2026
```

## Установка

```bash
cd app_admin
flutter pub get
flutterfire configure   # сгенерировать lib/firebase_options.dart под ваш проект
flutter run
```

> ⚠️ Файл `lib/firebase_options.dart` оставлен от исходного проекта.
> Для своего Firebase-проекта запустите `flutterfire configure` ещё раз.

## Защита от входа другой роли

Если в это приложение войдёт пользователь с ролью `customer` или `courier`,
экран авторизации покажет сообщение, что приложение предназначено для
администрации, с кнопкой «Выйти».

## Важно про Firestore Rules

Админ-панель читает коллекции `orders`, `users`, `complaints`, `dishes` целиком через
`StreamBuilder`. Убедитесь, что в Firestore Rules у пользователей с
`role == 'admin'` есть доступ на чтение всех документов, а также на запись в `dishes`.

Пример минимальных правил для коллекции `dishes`:

```
match /dishes/{dishId} {
  allow read: if request.auth != null;
  allow write: if request.auth != null
    && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
}
```
